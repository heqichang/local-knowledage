# Markdown 编辑器 - 验证清单

## 后端验证
- [ ] 验证 POST `/knowledge-bases/{kb_id}/notes` 端点能创建新的 Markdown 文档
- [ ] 验证创建的文档 `file_type` 为 "md"，状态初始为 "processing" 或 "pending"
- [ ] 验证空内容创建返回 400 错误
- [ ] 验证 GET `/knowledge-bases/{kb_id}/documents/{doc_id}/content` 能返回 md 文件内容
- [ ] 验证 PUT `/knowledge-bases/{kb_id}/documents/{doc_id}/content` 能更新文档内容
- [ ] 验证更新后 `file_hash` 发生变化
- [ ] 验证非 md 类型文档的 content 端点返回 400 错误
- [ ] 验证保存后后台任务正确触发（分段 + Embedding）
- [ ] 验证重新处理时旧的分段和向量被清除
- [ ] 验证文档列表能正确显示新创建和编辑后的笔记

## 前端验证
- [ ] 验证知识库文档列表页面有「新建笔记」按钮
- [ ] 验证 md 类型文档有「编辑」按钮，非 md 类型没有
- [ ] 验证路由 `/knowledge-bases/:kbId/notes/new` 可访问
- [ ] 验证路由 `/knowledge-bases/:kbId/notes/:docId/edit` 可访问
- [ ] 验证编辑器页面左栏是 CodeMirror，右栏是预览
- [ ] 验证输入 Markdown 时预览区实时更新
- [ ] 验证工具栏所有按钮可见且样式正确
- [ ] 验证格式按钮（加粗、斜体、标题等）能正确插入语法
- [ ] 验证撤销/重做按钮功能正常
- [ ] 验证视图切换（仅源码/仅预览/分屏）功能正常
- [ ] 验证保存按钮能正确保存文档
- [ ] 验证保存成功后有用户反馈（跳转或提示）
- [ ] 验证错误情况有友好的错误提示

## 功能验证
- [ ] 验证新建笔记保存后出现在文档列表中
- [ ] 验证编辑笔记保存后内容正确更新
- [ ] 验证代码块有语法高亮（测试 JavaScript、Python、TypeScript）
- [ ] 验证行内数学公式正确渲染
- [ ] 验证块级数学公式正确渲染
- [ ] 验证 GFM 特性（表格、任务列表、删除线等）正确渲染
- [ ] 验证新建的笔记能被搜索到
- [ ] 验证编辑后的笔记（内容变更）能被搜索到（旧内容搜不到，新内容能搜到）
- [ ] 验证笔记内容能参与 RAG 问答

## 代码质量验证
- [ ] 验证后端代码遵循项目现有的 Service 层模式
- [ ] 验证前端 API 函数和 hooks 遵循现有模式
- [ ] 验证 TypeScript 类型定义完整，无 `any` 类型
- [ ] 验证代码有适当的错误处理
- [ ] 验证没有引入不必要的依赖
